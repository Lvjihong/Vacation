# 文件处理命令  
## 命令格式与目录处理命令ls  
### 命令格式  
命令【-选项】【参数】  
- 选项是调整命令功能，就是让同一个命令有不同的功能  
- 参数是指操作对象  
加中括号说明可选
例：ls -la /etc  
说明：  
1. 个别命令使用不遵守此格式  
2. 当有多个命令时，可以写在一起  
3. 简化选项与完整选项  
-a等于--all  
### 目录处理命令：ls  
#### 命令名称：ls  
#### 命令英文原意：list  
#### 命令所在路径：/bin/ls  
#### 执行权限：所有用户  
#### 功能描述：显示目录文件  
#### 语法：ls 选项【-aldhi】【文件或目录】***ld，lh，li可连用***  
##### -a 显示所有文件，包括隐藏文件（在Linux中，·文件名，带点的是隐藏目录）  
##### -l 详细信息显示（目录下的详细数据）  
1. 第一部分：前十个字符
- 第一个：文件类型（-二进制文件，d目录，l软链接文件，最常见的就是这三个，其他一般用户不能操作）  
- 第二三四：指的是u所有者权限  
- 第五六七：指的是g所属组权限  
- 第八九十：指的是o其他人权限  
ugo的权限分为三类，r读；w写；x执行（最大权限，权限越小越好，够用就行，除非是脚本文件、命令什么的，一般没有x）
2. 第二部分：引用计数（对绝大部分人没用）。  
3. 第三部分：所有者（把所有用户分为三类：所有者u、所属组g、其他人o，文件和用户的关系，所有者只能是一个人；所有组只能是授权相同类型的一类用户）
4. 第四部分：所属组  
5. 第五部分：文件大小，默认单位字节，可以把-l改为-lh，人性化。  
6. 第六部分，最后修改时间。***在Linux中没有文件创建时间，只有最后访问时间、状态修改时间、文件数据修改时间，如果创建后没有修改过，访问时间就是创建时间***  
7. 第七部分：文件名。  
##### -d 查看目录属性  
-l -d 缩写为-ld，查看目录本身性质，而不是目录文件里的信息  
##### -i 查看i截点（每个文件的唯一标号）
## 目录处理命令  
### mkdir  
命令名称： mkdir  
命令英文原意：make directiories  
命令所在路径：/bin/mkdir  
执行权限：所有用户  
语法：mkdir -p【目录名】  
功能：  
1. 创建新目录：可同时创建多个  
2. -p递归创建：就是在一个没有的目录中创建目录，直接创建两个  
### cd  
命令名称：cd  
命令英文原意：change directory  
命令所在路径：shell内置命令  
执行权限：所有用户  
语法：cd[目录名]  
功能：切换目录  
范例：cd /tmp/Japan/boduo //切换到指定目录  
cd .. //回到上一级目录  
### pwd  
命令名称：pwd  
命令英文原意：print working directory  
命令所在路径：/bin/pwd  
执行权限：所有用户  
语法：pwd  
功能：显示当前目录  
### rmdir  
命令名称：rmdir  
命令英文原意：remove empty directories  
命令所在路径：/bin/rmdir  
执行权限：所有用户  
语法：rmdir【空目录】  
功能描述：删除空目录  
### cp  
命令名称：cp  
命令英文原意：copy  
命令所在路径：/bin/cp  
执行权限：所有用户  
语法：  
cp -rp 【原文件或目录】【目标目录】  
- cp -r 【原目录】【目标目录】（如果加一个新名字就是在复制的同时改名）：赋值目录  
- cp -p 保留文件属性（最后修改时间）  
功能描述：复制文件或目录  
### mv  
命令名称：mv  
命令英文原意：move  
命令所在路径：/bin/mv  
执行权限：所有用户  
语法：mv【原文件或目录】【目标目录】（这是切剪）；mv【原文件名】【新文件名】（这是改名***这个必须是在要改名的目录的目录下才行，否则得加路径***）  
功能描述：切剪文件、改名  
### rm  
命令名称：rm  
命令英文原意：remove  
命令所在路径：/bin/rm  
执行权限：所有用户  
语法：  
- rm -rf【文件】：删除文件  
- rm -rf【目录】：删除一个目录和里边的所有文件，不会一个文件一个文件问你是否要删除  
- rm -r【目录】：删除目录  
- rm -f【目录或者文件】：强制执行，不会问你是否删除  
功能描述：删除文件或目录

## 链接命令  
### 软链接  
软链接相当于Windows快捷方式  
#### 软链接性质
1. lrwxrwxrwx 这就是软链接，所有用户有所有权限，但是它不决定原文件性质，最终谁有什么权限还是取决于原文件  
2. 文件大小很小，因为它只是个符号链接  
3. 箭头指向源文件  
### 硬链接  
相当于cp -p，除了位置不同，其他信息全部相同，相比于软链接，如果删除源文件，软链接失效，硬链接不受影响  
#### 硬链接特点  
1. 相比于xp -p，它能够同步更新  
2. 能通过i节点识别，一个文件，只有一个i节点，但是一个i节点不一定对应一个文件  
3. 不能跨分区，软链接可以  
4. 不能针对目录使用，软链接可以

# 权限管理命令  
## 理解权限  
|代表字符|权限|对文件的含义|对目录的含义|
|:-:|:-:|:-:|:-:|
|r|读|查看文件内容|列出目录中的内容|
|w|写|修改文件内容|在目录中创建、删除文件|
|x|执行|执行文件|进入文件|
1. file:  
- r:cat/more/head/tail/less  
- w:vim  
- x:script/command  
2. directory:  
- r:ls  
- w:touch/mkdir/rmdir/rm  
- x:cd  
举例：root把一个目录权限是777，在里面建一个文件，文件权限是722，普通用户可以删除，哪怕他对该文件没有写的权对文件和对目录的权限一定得分清楚  
## 权限管理命令chmod  
命令名称：chmod  
命令英文原意：change the permissions mode of a file  
命令所在路径：/bin/chmod  
执行权限：所有用户  
功能描述：改变文件或目录权限  
语法：  
1. chmod [{u\g\o\a}{+\-\=}{rwx}]   [目录或文件]，多个权限改变，中间加逗号  
2. chmod 数字 文件名  
权限的数字表示：r=4,w=2,x=1  
3. chmod -R 再用1或者2，递归修改，修改目录及目录下所有文件和子目录的权限  
***更改文件权限可以是所有者，也可以是root***

语法：：umask[-S]  
-S以rwx的形式显示新建文件的缺省权限
